# テスト要件

## 最低テストカバレッジ：80%

テストの種類（すべて必須）：
1. **ユニットテスト** - 個別の関数、ユーティリティ、コンポーネント
2. **統合テスト** - APIエンドポイント、データベース操作
3. **E2Eテスト** - 重要なユーザーフロー（言語に応じてフレームワークを選択）

## テスト駆動開発

必須ワークフロー：
1. テストを先に書く（RED）
2. テストを実行する - 失敗するはず
3. 最小限の実装を書く（GREEN）
4. テストを実行する - 通過するはず
5. リファクタリングする（IMPROVE）
6. カバレッジを確認する（80%以上）

## テスト失敗のトラブルシューティング

1. **tdd-guide** エージェントを使用する
2. テストの独立性を確認する
3. モックが正しいか確認する
4. テストではなく実装を修正する（テストが間違っている場合を除く）

## エージェントサポート

- **tdd-guide** - 新機能に対してPROACTIVELYに使用し、テスト先行の作成を強制する
