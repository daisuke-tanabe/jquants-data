# エージェントオーケストレーション

## 利用可能なエージェント

| エージェント | 用途 | 使用タイミング |
|-------|---------|-------------|
| planner | 実装計画 | 複雑な機能、リファクタリング |
| architect | システム設計 | アーキテクチャの意思決定 |
| tdd-guide | テスト駆動開発 | 新機能、バグ修正 |
| code-reviewer | コードレビュー | コード記述後 |
| security-reviewer | セキュリティ分析 | コミット前 |
| build-error-resolver | ビルドエラー修正 | ビルド失敗時 |
| e2e-runner | E2Eテスト | 重要なユーザーフロー |
| refactor-cleaner | デッドコード削除 | コードメンテナンス |
| doc-updater | ドキュメント | ドキュメント更新 |
| rust-reviewer | Rustコードレビュー | Rustプロジェクト |

## エージェントの即時使用

ユーザーへの確認不要：
1. 複雑な機能リクエスト - **planner** エージェントを使用
2. 書いた/変更したコード - **code-reviewer** エージェントを使用
3. バグ修正または新機能 - **tdd-guide** エージェントを使用
4. アーキテクチャの意思決定 - **architect** エージェントを使用

## 並行タスク実行

独立した操作には常に並行タスク実行を使用する：

```markdown
# 良い例：並行実行
3つのエージェントを並行起動：
1. エージェント1：authモジュールのセキュリティ分析
2. エージェント2：キャッシュシステムのパフォーマンスレビュー
3. エージェント3：ユーティリティの型チェック

# 悪い例：不要な逐次実行
まずエージェント1、次にエージェント2、次にエージェント3
```

## 多角的分析

複雑な問題には、分割されたロールのサブエージェントを使用する：
- ファクトレビュアー
- シニアエンジニア
- セキュリティエキスパート
- 一貫性レビュアー
- 冗長性チェッカー
